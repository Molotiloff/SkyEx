# Документация по командам бота SkyEx

Ниже собраны все команды и обработчики, их синтаксис, назначение и ограничения по доступу.

## Базовые / справка
- `/start` — приветствие, регистрация чата в БД, создание базовых валют, подсказки по работе.
- `/помоги` — условия работы и правила (без показа клавиатуры).
- `/help` — краткий справочник по основным командам.
- `/кнопки` / `/скрыть` — показать/убрать основную клавиатуру.

## Калькулятор
- `/calc <выражение>` — посчитать выражение (поддерживаются `+ - * / () %`, дробные числа). Ошибки отображаются пользователю.
- `/` + выражение, если после слэша сразу идёт число/знак (например `/2+2`) — быстрый калькулятор без ключевого слова.
- Inline-режим: в любом чате введите `@имя_бота <выражение>` — выдаст варианты ответа, можно отправить как сообщение.

## Кошелёк клиента
- `/кошелек` — показать все счета чата с точностью балансов. Доступ: только менеджеры/админы.
- `/дай [ВАЛЮТА]` — показать ненулевые счета в компактном виде; с аргументом выводит конкретную валюту даже с нулём. Под сообщением — кнопки «Выписка за месяц/всё время».
- `/добавь КОД [точность]` — добавить валюту в кошелёк (точность 0–8). Доступ: менеджеры/админы.
- `/удали КОД` — удалить валюту из кошелька (с предупреждением, если баланс не ноль). Доступ: менеджеры/админы.
- `/<КОД> <сумма/expr> [комментарий]` — изменить баланс валюты (добавление/списание). Работает и из подписи к фото. Примеры: `/USD 250`, `/руб -1000 от сани`, `/usdt (500+200)*0.98`.
  - Доступ: менеджеры/админы; в отдельных «шумных» чатах команды игнорируются (ignore_chat_ids).
  - Если чат входит в `city_cash_chat_ids`, хвост парсится как `<точное имя клиента> [! комментарий]` и операция дублируется в чат клиента.
  - После операции выдаётся кнопка «Откатить изменение» (inline). Нажатие проводит обратную транзакцию (если не была отменена ранее).

## Короткие обменные команды (приём/отдача)
- Команды: `/пд`, `/пе`, `/пт`, `/пр`, `/пб`, `/прмск`, `/прспб`, `/прпер`, `/пп`.
- Формат: `/пд|/пе|/пт|/пр|/пб|/прмск|/прспб|/прпер|/пп <принимаем_expr> <од|ое|от|ор|об|ормск|орспб|орпер|оп> <отдаём_expr> [комментарий]`.
- Семантика: левая сумма — списываем у клиента, правая — зачисляем клиенту. Валюты мапятся внутри (`пд`→USD, `пе`→EUR, `пт`→USDT, `пр`→RUB, `пб`→USDW, `пп`→EUR500; для отдачи `од/ое/от/ор/об/ормск/орспб/орпер/оп`).
- Проверяется положительность сумм и расчёт курса. Если команда отправлена ответом на карточку бота — редактирует заявку, иначе создаёт новую.
- Доступ: менеджеры/админы; команды игнорируются в чатах из ignore_chat_ids.

## Полные заявки на наличные/FX
Обработчик `CashRequestsHandler` (доступ только менеджерам/админам). Общие правила:
- Город можно указать первым аргументом после команды (из списка настроенных городов), иначе берётся `default_city`.
- Контакты опциональны (0/1/2 аргумента в конце): `@username` или `+79991234567`. Комментарий отделяется `!`.
- Ответ на карточку бота с той же командой — редактирование заявки (валюты/тип менять нельзя).

### Депозит/выдача наличных
- Команды: `/депр`(RUB), `/депт`(USDT), `/депд`(USD), `/депе`(EUR), `/депб`(USDW), `/выдр`(RUB), `/выдт`(USDT), `/выдд`(USD), `/выде`(EUR), `/выдб`(USDW).
- Формат: `/депр [город] <сумма/expr> [кассир] [клиент] [! комментарий]`.
- Для `/выд*` порядок контактов тот же; суммы должны быть > 0. Если валюты нет в кошельке — попросит добавить через `/добавь`.

### Внутренний FX
- Команды: `/првд`(RUB→USD), `/пдвр`(USD→RUB), `/прве`(RUB→EUR), `/певр`(EUR→RUB), `/првб`(RUB→USDW), `/пбвр`(USDW→RUB), `/првп`(RUB→EUR500), `/ппвр`(EUR500→RUB), `/пдве`(USD→EUR), `/певд`(EUR→USD), `/пдвб`(USD→USDW), `/пбвд`(USDW→USD), `/пдвп`(USD→EUR500), `/ппвд`(EUR500→USD), `/певб`(EUR→USDW), `/пбве`(USDW→EUR), `/певп`(EUR→EUR500), `/ппве`(EUR500→EUR), `/пбвп`(USDW→EUR500), `/ппвб`(EUR500→USDW).
- Формат: `/команда [город] <сумма_in_expr> <сумма_out_expr> [кассир] [клиент] [! комментарий]`. Обе суммы > 0, валюты фиксированы командой.

### Тайминг заявки
- `/время HH:MM` — в чате заявок, ответом на сообщение с карточкой, устанавливает/обновляет строку времени в карточке и синхронизирует расписание для города. Доступ: менеджеры/админы.

## Клиентские карточки (ручная заявка)
- `/заявка <от_кого> <пд|пе|пт|пр> <expr> <од|ое|от|ор> <expr> [комментарий]`.
- Только в админском чате. Создаёт карточку без изменений баланса, отправляет её в чат заявок и копию в текущий чат.

## Балансы клиентов (агрегированно)
- `/бк` — показать все ненулевые балансы по всем клиентам.
- `/бк <ВАЛЮТА>` — балансы клиентов по валюте, |баланс| ≥ 1.
- `/бк <ВАЛЮТА> <+|->` — фильтр только положительные/отрицательные. Доступ: админские чаты из `admin_chat_ids`.

## Клиенты и города
- `/клиенты` — список активных клиентов с городами и chat_id. Доступ: админский чат.
- `/rmclient <chat_id>` — пометить клиента неактивным (is_active=false); требуется подтверждение по inline-кнопке. Доступ: админский чат.
- `/город <chat_id> <город>` — проставить/обновить город клиенту. Доступ: админский чат.

## Менеджеры
- `/mgr` — без аргументов выводит список менеджеров (user_id и отображаемое имя).
- `/mgr + <user_id> [display_name]` — добавить менеджера.
- `/mgr - <user_id>` — удалить менеджера.
- Работает только в одном настроенном админском чате (`admin_chat_id`).

## USDT-кошелёк (TRC-20)
- `/кош` — показать сохранённый адрес. Доступен всем.
- `/setwallet <адрес>` — задать/обновить адрес; лёгкая валидация длины. Доступ: админский чат.

## Рассылка(пока отключена, на особые ивенты)
- `/всем` — отправить всем активным клиентам картинку `BROADCAST_IMAGE_PATH` и текст `BROADCAST_ALL_TEXT`. Пропускает чаты без chat_id, учитывает блокировки. Доступ: менеджеры/админы.

## Диагностика
- `/whoami` — отладочная информация (user_id, имя, username, chat_id, название чата). Доступен всем.

## Офисные карточки
- `/екб`, `/члб` — отправляют карточку офиса с фото и инструкциями. Приоритет: готовый `file_id`, иначе загрузка локального файла из `images/`.
