# SkyEx Bot

Telegram-бот для внутреннего учета SkyEx: кошельки клиентов, обменные заявки, наличные заявки по городам, Google Sheets, AML-проверки, ордера по курсу, рассылки и служебные отчеты.

## Основные Возможности

- Учет клиентских кошельков по валютам: баланс, пополнение, списание, откат операции, выписки.
- Короткие обменные заявки `/пд ... ор ...` с карточками клиента и чата заявок.
- Наличные заявки `/деп*`, `/выд*` и FX-заявки `/прв*`, `/п*вр` по городам.
- Синхронизация заявок с Google Sheets: занесение сделки, удаление строк, чтение внутренних курсов.
- Расписание заявок по городам через `/время`.
- AML-проверка USDT/TRON адресов через GetBlock.
- Ордеры по курсу и live-обновление стакана Rapira/Grinex.
- Рассылка клиентам через `/всем`.
- Клиентская база, группы, менеджеры, офисные карточки и отчеты по балансам.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -U pip
# Установите зависимости проекта: aiogram, asyncpg, python-dotenv,
# google-api-python-client, google-auth, requests, beautifulsoup4, apscheduler и др.

cp .env.example .env  # если есть шаблон
python main.py
```

В production обычно запускать через `systemd`, `supervisor`, Docker или другой process manager:

```bash
python main.py
```

## Переменные Окружения

Обязательные:

- `BOT_TOKEN` — токен Telegram-бота.
- `DATABASE_URL` — строка подключения PostgreSQL.

Доступы:

- `ADMIN_CHAT_ID` — id админского чата.
- `ADMIN_IDS` — user_id админов через запятую, например `123,456`.

Заявки и города:

- `REQUEST_CHAT_ID` — общий чат заявок, legacy/fallback.
- `CASH_CHAT_ID` — карта городов в чаты заявок, например `екб:-1001,члб:-1002`.
- `DEFAULT_CITY` — город по умолчанию, например `екб`.
- `CITY_SCHEDULE_CHATS` — карта городов в чаты расписания.
- `CITY_CASH_CHAT_IDS` — id кассовых чатов через запятую.
- `SCHEDULE_CHAT_IDS` — чаты ежедневных отчетов по балансам.

Ордера и стаканы:

- `RATE_ORDERS_CHAT_ID` — чат ордеров по курсу.

Google Sheets:

- `GOOGLE_SHEET_URL` или `GOOGLE_SHEET_ID` — основная таблица.
- `SPREADSHEET_URL` или `SPREADSHEET_ID` — альтернативные имена для основной таблицы.
- `GOOGLE_CREDENTIALS_JSON` или `GOOGLE_SERVICE_ACCOUNT_JSON` — service account JSON inline.
- `GOOGLE_CREDENTIALS_FILE`, `GOOGLE_SERVICE_ACCOUNT_FILE` или `GOOGLE_APPLICATION_CREDENTIALS` — путь к service account JSON.
- `GOOGLE_SHEET_GID` — gid листа для публичного чтения балансов.
- `GOOGLE_BALANCE_CELLS_JSON` — карта ячеек курсов/балансов, например `{"USDT":"Главная!E8"}`.
- `GOOGLE_BALANCE_RANGE` — fallback-диапазон чтения балансов.

GetBlock AML:

- `GETBLOCK_IDENTITY`
- `GETBLOCK_PASSWORD`
- `GETBLOCK_LANG` — по умолчанию `en`.
- `GETBLOCK_USER_ID`
- `GETBLOCK_CURRENCY_CODE` — по умолчанию `TRX`.
- `GETBLOCK_TOKEN_ID` — по умолчанию `9`.
- `GETBLOCK_AML_PROVIDER` — по умолчанию `2`.
- `GETBLOCK_DIRECTION` — по умолчанию `2`.
- `GETBLOCK_SOURCE` — по умолчанию `1`.
- `GETBLOCK_TYPE` — по умолчанию `0`.
- `GETBLOCK_REPORTS_DIR` — по умолчанию `reports`.

## Доступы

- Команды менеджеров доступны пользователям из таблицы менеджеров, `ADMIN_IDS` или в разрешенных админских чатах.
- `/кош` для просмотра публичного USDT TRC-20 кошелька доступен всем.
- `/setwallet` доступен только в админском чате.
- `/всем` доступна только в админском чате.
- `/гарред` доступна только в админском чате.

## Команды Telegram

### Базовые

- `/start` — регистрация/обновление клиента и приветствие.
- `/помоги` — краткая справка.
- `/help` — список команд.
- `/кнопки` — показать основную клавиатуру.
- `/скрыть` — скрыть клавиатуру.
- `/whoami` — показать user_id/chat_id.

### Кошельки Клиентов

- `/кошелек` — показать все счета клиента.
- `/дай` — показать только ненулевые счета.
- `/дай <валюта>` — показать счет по конкретной валюте.
- `/добавь <валюта> [точность]` — добавить валюту клиенту.
- `/удали <валюта>` — удалить валюту с подтверждением.
- `/<валюта> <сумма|expr> [комментарий]` — изменить баланс.

Примеры:

```text
/кошелек
/дай
/дай USDT
/добавь USD 2
/добавь юсдт 0
/удали USD
/USD 250
/RUB -100
/USDT (2+3*4)
/дол 1000 оплата
```

В кассовых чатах можно указывать клиента для дублирования операции:

```text
/USD 1000 Иван Иванов
/RUB -50000 Иван Иванов ! комментарий
```

### Публичный USDT Кошелек

- `/кош` — показать USDT TRC-20 кошелек.
- `/setwallet <адрес>` — задать USDT TRC-20 кошелек в админском чате.

### Короткие Обменные Заявки

Формат:

```text
/<что получаем> <сумма|expr> <что отдаем> <сумма|expr> [комментарий]
```

Получаем:

- `/пд` — USD
- `/пе` — EUR
- `/пт` — USDT
- `/пр` — RUB
- `/пб` — USDW
- `/пп` — EUR500
- `/прмск` — RUB Moscow
- `/прспб` — RUB SPB
- `/прпер` — RUB Perm

Отдаем:

- `од` — USD
- `ое` — EUR
- `от` — USDT
- `ор` — RUB
- `об` — USDW
- `оп` — EUR500
- `ормск` — RUB Moscow
- `орспб` — RUB SPB
- `орпер` — RUB Perm

Примеры:

```text
/пд 1000 ор 91000 Клиент Петров
/пр 100000 от 100000/94 срочно
/пт 700 од 700*1.08
/пб 5000 от 5000/1.02
```

Поведение:

- В чате клиента создается карточка с клиентским номером заявки.
- В чате заявок создается карточка с номером заявки и номером для таблицы.
- В чате заявок доступна кнопка `Занести в таблицу`.
- В чате клиента доступна отмена.
- При редактировании нужно ответить новой командой на карточку бота.
- При отмене бот возвращает балансы; если заявка была занесена в таблицу, предложит удалить строки из Google Sheets.

### Наличные Заявки

Формат:

```text
/<команда> [город] <сумма|expr> [контакт 1] [контакт 2] [! комментарий]
```

Депозит:

- `/депр` — RUB
- `/депт` — USDT
- `/депд` — USD
- `/депе` — EUR
- `/депб` — USDW

Выдача:

- `/выдр` — RUB
- `/выдт` — USDT
- `/выдд` — USD
- `/выде` — EUR
- `/выдб` — USDW

Примеры:

```text
/депр 1000
/выдр 1000
/депт екб 100
/выдд члб 500 Иван Петров
/депр члб (50000+10000) Кассир Клиент ! срочно
```

### FX Заявки Наличными

Формат:

```text
/<команда> [город] <сумма_in> <сумма_out> [Кассир] [Клиент] [! комментарий]
```

Команды:

- `/првд` — RUB -> USD
- `/пдвр` — USD -> RUB
- `/прве` — RUB -> EUR
- `/певр` — EUR -> RUB
- `/првб` — RUB -> USDW
- `/пбвр` — USDW -> RUB
- `/првп` — RUB -> EUR500
- `/ппвр` — EUR500 -> RUB
- `/пдве` — USD -> EUR
- `/певд` — EUR -> USD
- `/пдвб` — USD -> USDW
- `/пбвд` — USDW -> USD
- `/пдвп` — USD -> EUR500
- `/ппвд` — EUR500 -> USD
- `/певб` — EUR -> USDW
- `/пбве` — USDW -> EUR
- `/певп` — EUR -> EUR500
- `/ппве` — EUR500 -> EUR
- `/пбвп` — USDW -> EUR500
- `/ппвб` — EUR500 -> USDW

Примеры:

```text
/првд члб 100000 1000 Кассир Клиент
/пдвр 1000 91000
/пдве 1000 920
```

### Работа Со Временем И Сделкой

- `/время HH:MM` — ответом на карточку заявки добавляет/заменяет время в карточке и синхронизирует расписание.
- Кнопка `Выдано` — проводит депозит/выдачу по клиентскому кошельку.
- Кнопка `Сделка завершена` — помечает сделку в чате заявок и убирает из расписания.
- Кнопка `Сделка отменена` — помечает отмену и убирает из расписания.

Пример:

```text
/время 10:30
```

### Google Sheets

- Кнопка `Занести в таблицу` в чате заявок записывает сделку в Google Sheets.
- Для заявок `accept_short` payload хранится в БД, поэтому после рестарта бот может восстановить данные по номеру таблицы.
- При отмене заявки, уже занесенной в таблицу, бот спросит, удалять ли строки из таблиц.
- RUB-регионы (`RUB`, `РУБМСК`, `РУБСПБ`, `РУБПЕР`, `РУБТЮМ`) приводятся к `RUB` при записи в таблицу.

### AML

- `/амл <TRON_USDT_адрес>` — поставить AML-проверку в очередь.


### Ордера И Курсы

- `/ордер <курс>` — создать ордер на курс.
- `/курс <комиссия>` — ответом на сообщение ордера активировать отслеживание.
- `/гар` — показать глубину asks.
- `/гар-` — показать первый bid.
- `/гарред` — создать/обновлять live-сообщение стакана в админском чате.

Примеры:

```text
/ордер 91.5
/курс -0.5
/курс +0.3
/гар
/гар-
/гарред
```

### Клиенты, Группы, Менеджеры

- `/клиенты` — список активных клиентов.
- `/клиенты <группа>` — список клиентов группы.
- `/rmclient <chat_id>` — мягко удалить клиента с подтверждением.
- `/группа <chat_id> <группа>` — назначить клиенту группу.
- `/mgr` — список менеджеров.
- `/mgr + <user_id> [имя]` — добавить менеджера.
- `/mgr - <user_id>` — удалить менеджера.

Примеры:

```text
/клиенты
/клиенты VIP
/rmclient 123456789
/группа 123456789 VIP
/mgr
/mgr + 123456789 Константин
/mgr - 123456789
```

### Отчеты И Балансы

- `/бк` — все ненулевые балансы клиентов.
- `/бк <валюта>` — клиенты с балансом по валюте, фильтр `|баланс| >= 1`.
- `/бк <валюта> +` — клиенты с положительным балансом.
- `/бк <валюта> -` — клиенты с отрицательным балансом.
- Ежедневные отчеты отправляются в `SCHEDULE_CHAT_IDS`.

Примеры:

```text
/бк
/бк USDT
/бк RUB +
/бк USD -
```

### Рассылка

- `/всем` — начать рассылку всем активным клиентам.
- `/всем <группа>` — рассылка клиентам указанной группы.

Сценарий:

1. В админском чате отправить `/всем` или `/всем VIP`.
2. Ответить на сообщение бота текстом, фото или альбомом.
3. Проверить preview.
4. Нажать `Отправить рассылку` или `Отменить рассылку`.

### Админская Ручная Заявка

- `/заявка <от_кого> <пд|пе|пт|пр> <сумма/expr> <од|ое|от|ор> <сумма/expr> [комментарий]`

Примеры:

```text
/заявка Клиент_Иванов пр 100000 от 100000/65 Приоритет
/заявка Петров /пт (700+300) /од 1000 срочно
```

### Офисы

- `/екб` — карточка офиса Екатеринбург.
- `/члб` — карточка офиса Челябинск.
- `/тюм` — карточка офиса Тюмень.

### Калькулятор

- `/calc <выражение>` — посчитать выражение.
- `/+100-5%` и похожие slash-выражения — быстрый расчет.
- Inline-калькулятор работает через `@bot <выражение>`.

Примеры:

```text
/calc (2+3)*100-50%
/1000-5%
```

## Архитектура

- `handlers/` — Telegram routing, auth, callback orchestration.
- `services/` — бизнес-сценарии и use-cases.
- `services/exchange/` — короткие обменные заявки.
- `services/cash_requests/` — наличные заявки, расписания, выдача, завершение/отмена.
- `services/wallets/` — кошельки: parser/query/mutation/undo/text/interaction.
- `services/broadcast/` — рассылки: service/session/preview.
- `services/rate_order/` — ордера и стаканы.
- `services/aml/` — GetBlock AML.
- `db_asyncpg/repositories/` — специализированные репозитории.
- `db_asyncpg/repo.py` — совместимый фасад над репозиториями.
- `gutils/` — Google Sheets integration.
- `utils/` — общие утилиты и compatibility wrappers.
