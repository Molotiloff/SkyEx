# SkyEx Bot

Телеграм‑бот для внутренней бухгалтерии: учёт кошельков клиентов, быстрые обменные заявки, кассовые операции по городам и синхронизация с Google Sheets.

## Возможности
- Управление кошельками клиентов (балансы, добавление/удаление валют, откат операций).
- Короткие команды приёма/выдачи валют и заявки наличными/FX с поддержкой городов.
- Агрегированные отчёты по балансам клиентов.
- Рассылка клиентам готового объявления.
- Карточки офисов (фото + инструкция), калькулятор и вспомогательные команды (`/whoami` и др.).
- Интеграция с Google Sheets для отметок «Занесено в таблицу» / удаления строк.

Полное описание синтаксиса команд см. в `COMMANDS.md`.

## Настройки окружения
Переменные читаются из окружения (при наличии `python-dotenv` — и из `.env`):

- `BOT_TOKEN` — токен Телеграм-бота (обязательно).
- `DATABASE_URL` — строка подключения Postgres (обязательно).
- `ADMIN_CHAT_ID` — id админского чата (доступ к большинству команд).
- `ADMIN_IDS` — список user_id админов через запятую (например `123,456`).
- `REQUEST_CHAT_ID` — общий чат заявок (опционально, legacy).
- `CASH_CHAT_ID` — карта город→чат заявок, формат `екб:-100,члб:-200`.
- `CITY_SCHEDULE_CHATS` — карта город→чат расписания, формат аналогичен `CASH_CHAT_ID`.
- `DEFAULT_CITY` — город по умолчанию (строка, напр. `екб`).
- `CITY_CASH_CHAT_IDS` — id кассовых чатов (через запятую) для дублирования операций клиентам.
- Переменные для Google Sheets (если используете отметки «Занесена в таблицу»):
  - `GOOGLE_SHEET_URL` или `GOOGLE_SHEET_ID`
  - файл сервисного аккаунта (`exchange-bot-…json`) должен быть доступен в рабочей директории или указан через `GOOGLE_APPLICATION_CREDENTIALS`.

## Структура
- `handlers/` — обработчики команд (кошельки, заявки, офисы, кросс-курс, calc и т.п.).
- `services/cash_requests/` — логика заявок наличными/FX и расписаний.
- `utils/` — утилиты (аутентификация, форматирование, парсинг, интеграции с Sheets).
- `db_asyncpg/` — работа с Postgres (репозиторий/пул подключений).
- `COMMANDS.md` — полная документация по телеграм-командам.

## Запуск в продакшене
- Настройте `BOT_TOKEN` и `DATABASE_URL`.
- Добавьте Google Service Account в редакторы нужной таблицы (для функций таблиц).
- Обеспечьте логирование stdout/stderr или настройте `logging` по своим требованиям (см. `bot_app.run_app`).
- Запускайте через процесс‑менеджер (systemd/supervisor/docker) командой `python main.py`.
